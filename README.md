# Healthreports

**当一场大病降临：14亿人的健康账本**  
一份 AI 辅助生成的健康数据报告（含可视化图表与政策解读，单文件 HTML，零外部依赖）。

## 在线阅读
👉 https://dingchenchen6.github.io/Healthreports/

---

## 报告内容
- **多章节结构**：从短视频“求助/自救”现象切入，延伸到重大疾病负担、医保制度、健康不平等与生命观讨论  
- **数据可视化**：纯 CSS/SVG 组件化图表（条形图、对比表、概念示意图等），适配移动端与微信内置浏览器  
- **参考文献**：官方统计、政策文件、学术研究与权威媒体报道（集中列在文末，便于核对）

---

## 技术特点
- ✅ **零外部依赖**：不加载任何 CDN/外部脚本，国内访问更稳定  
- ✅ **移动端友好**：针对手机浏览与长文阅读做了排版优化  
- ✅ **单文件部署**：核心内容在 `index.html`，直接用 GitHub Pages 发布  
- ✅ **图表自包含**：不依赖 Chart.js 等库，便于长期维护与迁移

## 数据来源（概述）
本报告主要整理自公开渠道数据与报告，包括但不限于：  
- 国家卫生健康委员会、国家统计局、国家医保局等官方发布  
- 国家癌症中心及权威研究/报告  
- 公开的行业报告与可信媒体梳理材料  

> 说明：文末“参考文献/数据来源”列出更完整的条目与出处信息，供核验与延伸阅读。

## 使用与部署（GitHub Pages）
1. 将仓库根目录的主页面命名为 **`index.html`**  
2. GitHub 仓库 → **Settings** → **Pages**  
3. Source 选择 **Deploy from a branch**，分支选 `main`，目录选 `/ (root)`  
4. 保存后访问：`https://<username>.github.io/<repo>/`

---

## 更新方式（推荐）
- 只维护一个 `index.html` 为主版本  
- 历史备份建议放到 `archive/` 目录或删除，避免 Pages 入口混乱  
- 每次更新后提交 commit，GitHub Pages 会自动刷新
